# sync-calendar — Android (Kotlin)

Настоящее нативное Android-приложение. Тот же протокол, что и в
Python-прототипе (`devices/*.json`, `reminders/*.json`), но теперь —
с настоящим `AlarmManager`, который умеет будить телефон по расписанию,
даже когда приложение закрыто.

Компилировать самому не нужно вообще ничего — за тебя это сделает
GitHub Actions в облаке. Ниже — по шагам.

## 1. Заведи репозиторий на GitHub

Если аккаунта нет — регистрация бесплатная, на github.com.
Создай новый репозиторий (кнопка **New**), можно приватный.

## 2. Залей туда файлы из этого архива

Самый простой способ — без командной строки:
на странице пустого репозитория нажми **uploading an existing file**,
перетащи туда всё содержимое этого архива (сохраняя структуру папок:
`app/`, `.github/`, `settings.gradle.kts` и т.д. — то есть распакуй
архив и перетащи содержимое папки, а не сам .zip) и нажми **Commit**.

Если умеешь git — можно и им:
```
git init
git add .
git commit -m "init"
git remote add origin <ссылка на твой репозиторий>
git push -u origin main
```

## 3. Сборка запустится сама

Как только файлы окажутся в репозитории, GitHub Actions сам увидит
`.github/workflows/build-apk.yml` и запустит сборку. Открой вкладку
**Actions** вверху репозитория — там будет жёлтый кружок (идёт сборка),
потом зелёная галочка (готово) минуты через 2–3.

## 4. Скачай APK

Зайди в завершившийся прогон (клик по нему), внизу страницы —
раздел **Artifacts**, там `sync-calendar-debug-apk`. Скачай — это .zip,
внутри — `app-debug.apk`.

Если открываешь страницу Actions прямо с телефона — можно скачать
сразу на телефон и не пересылать отдельно.

## 5. Установи на телефон

Перенеси APK на телефон (если собирал не на нём), открой файл —
Android спросит разрешение на установку из этого источника (для
файлового менеджера/браузера, через который открываешь) — разреши
и установи. Это не Play Store, поэтому появится стандартное
предупреждение «из непроверенного источника» — это нормально для
sideload, не ошибка.

## 6. Первый запуск

- Спросит разрешение на уведомления — разреши, иначе будильники
  будут срабатывать без звука/баннера.
- Спросит имя устройства — как оно будет видно в списке на других
  устройствах.
- Спросит папку — выбери именно ту, что синхронизирует Syncthing.

Дальше — список напоминаний, кнопка «+» внизу справа для добавления,
с выбором даты, времени и того, на каких устройствах это должно
прозвенеть.

## Почему подпись одна и та же от сборки к сборке

В архиве лежит `app/debug.keystore` — обычный debug-ключ, специально
зафиксированный и закоммиченный (для debug-ключей это нормальная
практика, в отличие от боевых release-ключей). Без этого каждая
сборка в CI получала бы новый случайный ключ, и телефон отказывался
бы ставить новую версию поверх старой без полного удаления приложения.

## USE_EXACT_ALARM — почему не нужно ходить в настройки

Обычно точные будильники на Android 14+ требуют, чтобы пользователь
вручную включил разрешение в настройках. Но `USE_EXACT_ALARM` —
отдельное разрешение именно для календарных/будильниковых приложений,
которое система выдаёт автоматически при установке. Это не политика
Google Play (мы туда и не идём) — это поведение самой ОС, работает
одинаково что при sideload, что при установке из магазина.
